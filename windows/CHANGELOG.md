# Windows Changelog

## 2.2.8 — 2026-07-18

- 公开项目与下载包统一显示 `Codex Theme Switch · Codex 主题衣橱`，标明由 `myxsf` 维护且目前暂无赞助商；CI 阻止旧赞助链接回流。
- Windows 无交互或精简用户环境无法返回系统桌面目录时，快捷方式安装与卸载回退到 `%USERPROFILE%\Desktop`，不再因空路径中止。
- 恢复八套主题的 Codex 原生项目栏，主题边框只包裹真实输入框，不再遮挡项目选择器。
- 高窗口使用完整 Hero 安全区与真实功能卡利用高度，输入框不再被拉高，底部留白统一受 12% 验收约束。
- 清透与黑金复合海报的烘焙假卡片使用主题渐隐排除，避免与真实 Codex 功能卡混淆。
- 侧栏拖动时视觉卡片按 `ResizeObserver` 和动画帧实时同步，避免慢半拍或穿入侧栏。
- 与 macOS 共享项目栏可见、组件净距、输入框高度和纵向溢出回归断言。
- 黑金主题高窗口卡片高度恢复为 298px，保持输入框原生高度并把底部留白压到 12% 以内。

## 2.2.7 — 2026-07-18

- 八套图库主题统一为自然流式输入区，消除卡片重叠、输入框沉底和异常纵向空隙。
- 粉系与红白科幻 Hero 文案增加明确前景和可读性光晕；黑金输入框统一使用高对比暖白文字。
- 侧栏任务标题持续预留原生操作按钮空间，长标题省略且按钮只在悬停或聚焦时显示。
- 新增共享 CSS 回归断言，macOS 实时深检同时约束卡片净距、纵向溢出和输入框视口边界。

## 2.2.6 — 2026-07-18

- ENFP、紫夜限定和初音未来使用独立 visual schema、卡片文案、侧栏身份和主题便签。
- 主题卡片连接真实 Codex 输入框或插件入口，支持键盘操作且不会自动发送。
- 未声明时不再创建通用粉紫闪点、蝴蝶结和拍立得，避免污染初音与其他主题。
- 修复输入框区域 flex-grow 造成的沉底、巨大空白和窗口放大后错位。
- Hero 改为按宽高比响应式缩放，矮窗口进一步压缩 Hero、卡片与输入区高度。
- 紫夜改为浅紫主工作区和浅色侧栏，恢复原型对比度；最近任务长标题不再与操作图标重叠。
- 隐藏原型不要求的项目选择器行，并压缩卡片到输入框的纵向空隙。
- 切换主题时清理旧 chrome、旧卡片和旧布局状态。

## 2.2.5 — 2026-07-18

- 与 macOS 统一发布版本；QQ 首次进入恢复参考侧栏比例，随后保留原生分隔器拖动并实时同步主工作区。
- 限制侧栏真实内容层宽度并清除线程容器横向位移，避免切页后的文本泄漏和主内容裁切。

- 新增 10 组缩放几何自测与窗口 smoke 断言，覆盖北/西固定对边、最大化和精确还原。

## 2.0.0 — 2026-07-17

### 新增

- 原生 WPF「Codex 主题衣橱」，支持触摸、拖动、横向滚轮和吸附动画。
- 读取共享 `themes/catalog.json`，展示全部启用的 Windows 主题与原皮。
- 原皮模式、主题状态命令、原子串行切换、失败自动回滚。
- Windows ZIP、SHA-256、静态测试和 WPF 编译测试。

### 改进

- 优先复用官方 Codex 内置 Node.js，减少外部运行依赖。
- Codex 包与可执行文件动态发现，不再按进程名误杀其它 ChatGPT 程序。
- CDP 显式绑定 `127.0.0.1`，校验 WebSocket 地址并探测真实 Codex DOM。

### 修复

- 最大化前记录正常窗口边界，还原时恢复实际尺寸，避免只切换 `windowState` 后窗口异常。
- 恢复操作现在读取已保存端口并完整验证注入标记、样式和装饰 DOM 均已移除。
